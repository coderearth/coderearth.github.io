Installing and Running Hadoop 2.5 on Windows 7

We’re going to see how to install and run a single node Hadoop cluster, version 2.5 on a 64-bit Windows 7. For ease of reading, I have divided this tutorial into multiple chapters.

Prerequisites

  • Do not use Cygwin.
  • You need to have admin privileges to your computer.
  • Follow the exact versions of various softwares with the exact installation paths that are mentioned below.
  • You need a good internet connection, as quite a bit of downloading is involved.

Chapter 1: Install JAVA & Configure PATH

  1. Download JDK 1.7 and install it in C:\Java\jdk1.7.0_71.
  2. Set the JAVA_HOME path variable.
    1. Right-click the My Computer icon on your desktop and select Properties.
    2. Click the Advanced system settings on the left pane.
    3. Click the Environment Variables button.
    4. Under System Variables, click New.
    5. Enter the variable name as JAVA_HOME.
    6. Enter the variable value as C:\Java\jdk1.7.0_71.
    7. Click OK
  3. Add %JAVA_HOME%\bin to Path
    1. Under System Variables, find the Path variable.
    2. Click Edit
    3. Prepend %JAVA_HOME%\bin;
    4. Click OK

  4. Verify the JAVA setup.

Congratulations, you can now proceed to Chapter 2: Preparing Environment.

Chapter 2: Preparing Environment

Part 1: Download & Extract Hadoop Source

  1. I use ConEmu as my console, but the windows command prompt should suffice, as well.
  2. We will be needing 7zip for extracting some archives later. Download & Install it if you don’t have already.
  3. Download the Hadoop 2.5.2 stable source from here.
  4. Extract the .tar.gz file with 7zip twice to get the hadoop-2.5.2-src folder.
  5. Let’s copy the contents of the above folder to C:\hadoop.

At this stage, we have the hadoop source. We now have to build hadoop - to do this, we have to install some dependencies.

Part 2: Installing Windows Dependencies

Dependency 1: Apache Maven
  1. Download Apache Maven 3.2.3
  2. Just like we did with JAVA, Set the M2_HOME path variable as C:\apache-maven-3.2.3 and prepend %M2_HOME%\bin; to the system Path variable.

  3. Verify maven installation by typing mvn --version in a command prompt.

Dependency 2: Protocol Buffers 2.5.0

Protocol Buffers is Google’s efficient data interchange format. Read more here.

  1. Download Protocol Buffers 2.5.0
  2. Extract it to C:\protoc-2.5.0-win32 and prepend it to the system Path variable.
  3. Verify that protoc.exe is in the Path.

Dependency 3: Windows SDK

Download and install the Windows SDK from here. Once it is downloaded and installed, it will open a Help Manager window, where in select only Windows SDK to be updated.

Dependency 4: Unix command-line tools

Specifically, we need Unix command-line tools from GnuWin32 or Cygwin: sh, mkdir, rm, cp, tar, gzip. Since however we won’t be using Cygwin, we’ll use GnuWin32.

  1. Download GetGnuWin32 from here.
  2. Please skip to step 5. Steps 3 & 4 are further information for the curious & the adventurous.
  3. Note that GetGnuWin32 is not GnuWin32 itself, but a package maintenance utility. More on that here, if you’re interested to read that is.
  4. Since downloading from the GetGnuWin32 might take a while, and if you’re feeling miserly about your bandwidth usage, and if you know what you’re doing, you can skip this section altogether provided you have done either of the below:
    1. If you have Cygwin, add its bin to Path, after making sure it has the above mentioned commands.
    2. Individually download the above commands from GnuWin32 and add them to your Path.
  5. Click install on the GetGnuWin32 window, by default it should go to your Downloads folder.
  6. From a command prompt, go where GetGnuWin32 was installed to, in my case, C:\Users\suhas\Downloads\GetGnuWin32 and run download.bat. This step will take a while.
  7. Run install C:\gnuwin32.
  8. Append C:\gnuwin32\bin to your Path.
Dependency 5: Zlib 1.2.8

To be fair, zlib comes as part of the above GnuWin32 package, but let’s explicitly set this up as Hadoop 2.5 plays nicely with zlib 1.2.7 and 1.2.8 versions.

  1. Download Zlib compiled dll package from here, if that link doesn’t work, try here.
  2. Extract it to C:\zlib-1.2.8
  3. Add ZLIB_HOME with value as C:\zlib-1.2.8 to System Variables, and then append %ZLIB_HOME%; to Path.
Optional Dependency 6: Findbugs 1.3.9
  1. Download Findbugs 1.3.9 from here.
  2. Extract to C:\findbugs-1.3.9.
  3. Under Environment Variables, add a FINDBUGS_HOME system variable with value as C:\findbugs-1.3.9.
  4. Append %FINDBUGS_HOME%\bin; to Path.

Part 3: Final Preparation To Build

  1. Add another System Variable under Environment Variables with the name Platform with value set to x64. This is case-sensitive. It’s not PLATFORM or platform, but Platform.
  2. Build must be run through Windows SDK Command Prompt (Start > All Programs > Microsoft Windows SDK v7.1 > Windows SDK 7.1 Command Prompt).


Congratulations! You are now ready to build Hadoop. Move on to Chapter 3: Building Hadoop from Source.
